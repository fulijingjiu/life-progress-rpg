# 目标用户与待验证任务

> 状态：研究假设，不是真实访谈结论
> 最近校准：2026-07-24

本文不使用虚构姓名、职业、收入和设备来制造确定感。v0.1 先验证用户任务、使用情境和内容价值；人口属性只有在研究数据表明存在显著差异时再用于细分。

## 1. 首要目标用户

首轮聚焦：

> **愿意了解自己的近期状态，偶尔尝试记录，但因为输入负担高、记录后没有反馈而难以持续的成年人。**

核心待办（JTBD）：

```text
当一天结束、我想知道最近过得怎么样时，
我希望用很少的操作留下今天的状态，
并马上得到一个具体、不过度解读的回应，
这样以后回看时能发现对自己有用的线索。
```

这一定义不预设用户喜欢 RPG、愿意分享、信任 AI 或存在“时间焦虑”。这些都必须单独验证。

## 2. 优先情境

| 情境 | 当前替代方式 | 主要阻力 | 产品机会 |
|------|--------------|----------|----------|
| 睡前快速回顾 | 不记录、备忘录、聊天窗口 | 不想写长日记 | 60 秒记录 + 当日回应 |
| 状态持续波动 | 凭印象回忆 | 容易只记住极端时刻 | 有日期和证据的个人回顾 |
| 想了解某个习惯 | 表格、习惯 App、主观猜测 | 记录分散，难以比较 | 用户主动选择观察问题 |
| 想保存生活片段 | 相册、朋友圈、手账 | 信息零散或公开压力 | 私密、可导出的成长档案 |

## 3. 原型用户组

### A. 低负担记录者——v0.1 首要对象

**现状**

- 想记录，但无法稳定写长文。
- 可以接受选择心情、能量和标签。
- 记录后如果没有结果，会觉得这件事没有意义。

**需要的内容**

- 保存后立即出现一条有事实依据的回应。
- 没写备注时也能得到自然但不过度解读的反馈。
- 错过一天不被惩罚或羞辱。

**留存条件**

- 第二次打开比第一次更省力。
- 回应不重复、不鸡汤。
- 一段时间后能获得新的回看价值。

### B. 自主探索者——周期内容候选对象

**现状**

- 对自己的精力、情绪或生活节奏有具体疑问。
- 愿意连续记录一段时间换取更清晰的回顾。
- 对没有依据的 AI 结论比较敏感。

**需要的内容**

- 明确样本数、日期范围和缺失数据。
- 能展开查看支撑结论的原始记录。
- 自己选择要观察的问题，而不是接受“人生建议”。

**留存条件**

- 新分析确实来自累积数据。
- 产品会说“不知道”，允许纠正和关闭。

### C. 视觉记录者——体验质量敏感对象

**现状**

- 已有手账、相册或日记习惯。
- 在意排版、回顾质感和内容保存。
- 不愿为了游戏机制牺牲记录真实性。

**需要的内容**

- 清晰、克制、适合长期阅读的呈现。
- 月度片段或个人故事能编辑、保存和导出。
- 分享必须由用户主动触发，并可隐藏敏感字段。

**留存条件**

- 产品不是模板化数据面板。
- 长文本、空状态和历史回看仍有质感。

## 4. 暂不优先的用户

| 用户需求 | 暂不优先原因 |
|----------|--------------|
| 主要追求排行榜、组队和竞争 | 与私人、自主、非比较定位冲突 |
| 需要 AI 给出医疗或心理诊断 | 高风险，超出产品能力 |
| 需要 AI 替自己规划人生或做重大决定 | 不可信且削弱用户自主 |
| 只想要完整任务管理 RPG | Habitica 等产品已经覆盖，偏离记录核心 |
| 只因死亡倒计时寻求刺激 | 可能看一次即走，并有焦虑风险 |

“不优先”不代表排斥这些人，而是 MVP 不为这些任务增加范围。

## 5. 用户旅程假设

### 首次体验

```text
看到人生进度概念
  → 理解它是估算且可关闭
  → 60 秒完成第一次记录
  → 收到具体的当日回应
  → 判断“是否值得再记一次”
```

人生进度负责吸引注意，当日回应负责完成价值确认。

### 短期复访

```text
再次打开
  → 当天输入比第一次更快
  → 回应没有重复
  → 历史开始形成可回看的片段
```

### 周期价值

```text
记录达到门槛
  → 用户主动打开周期回顾
  → 查看事实、证据和不确定性
  → 选择一个想继续观察的问题
  → 后续记录更有目的
```

周期价值属于 v0.2 以后实验，不能在 v0.1 宣传为已上线。

## 6. 关键研究问题

优先验证：

1. 人生进度带来的是好奇、掌控还是焦虑？
2. 用户愿意长期填写的最少字段是什么？
3. 第一次回应是否具体到值得第二次记录？
4. 用户认为“有帮助”的回应究竟包含什么？
5. 用户能否区分当日回应和长期趋势？
6. 展示证据是否提高信任，还是增加理解负担？
7. 哪些用户愿意启用 AI？他们担心发送哪些数据？
8. 用户更愿意回看事实、故事、图表还是观察问题？

避免诱导式问题，例如“你喜欢这个很酷的 AI 洞察吗？”

## 7. 招募与样本

首轮建议招募 12～18 人，至少覆盖：

- 当前完全不记录
- 偶尔使用备忘录或聊天窗口记录
- 已使用情绪追踪或日记产品
- 对 AI 积极、中立和拒绝的用户
- 看到人生进度后感到好奇、中性和不适的用户

先按行为和态度分析，不用职业、城市或收入替代真实差异。

小样本报告原始人数、任务完成情况和逐条观察，不展示伪精确百分比。

## 8. 验证任务

### 可用性任务

1. 不经讲解完成首次引导。
2. 记录一个“心情不错但很累”的真实或模拟日期。
3. 解释回应中哪句话来自自己的输入。
4. 关闭人生进度后继续记录。
5. 找到、编辑、导出并删除一条记录。

### 内容评测

为同一记录准备三类回应：

- 机械复述
- 泛化鼓励
- 具体事实回应

让参与者分别评价：

- 哪一条最像理解了记录
- 哪一条最可信
- 哪一条让人想继续记录
- 哪一条令人不适或被评判

## 9. 成功与停止信号

正向信号：

- 用户无需解释即可完成记录。
- 多数参与者能指出回应的事实依据。
- 用户愿意为“以后看到个人线索”再记录一次。
- 拒绝 AI 的用户仍认为本地功能完整。

停止或调整信号：

- 人生进度成为主要焦虑来源。
- 回应经常被评价为鸡汤、重复或越权。
- 用户认为必须提交隐私内容才能获得价值。
- 周期分析无法说明数据依据。
- 主要使用动机变成连续签到或虚拟奖励，而不是自我理解。
