# 产品概念

> 状态：产品方向；v0.1 范围以 [MVP 规划](../planning/milestone-v1.md) 为准

## 一句话定义

**用一个有辨识度的人生进度入口，让轻量记录立即得到具体回应，并把累积记录逐渐变成可核对的个人线索。**

## 核心判断

产品的卖点不是功能数量，也不是“AI 很懂你”：

- 人生进度负责第一次注意和传播。
- 60 秒记录负责降低开始成本。
- 当日回应负责第一次价值确认。
- 有证据的周期分析负责长期留存。
- 本地优先、可关闭、可纠正和可导出负责建立信任。
- 视觉质量负责让内容更容易理解，而不是包装空洞结论。

详细内容模型见[内容价值与个人分析策略](./content-strategy.md)。

## 要解决的问题

| 待验证问题 | 当前方案 | 主要风险 |
|------------|----------|----------|
| 时间流逝缺少感知 | 可关闭、明确标注为估算的进度展示 | 引发时间或死亡焦虑，只形成一次性好奇 |
| 记录负担过高 | 60 秒完成心情、能量、标签和可选备注 | 过度简化后没有回看价值 |
| 记录后没有反馈 | 本地当日回应；可选 AI 文字整理 | 机械复述、鸡汤、重复或越权 |
| 长期数据难回顾 | 达到门槛后生成有证据的周期内容 | 小样本和相关性被误解成结论 |
| 私密内容缺少信任 | 本地优先、明示同意、可导出删除 | 宣传与真实数据行为不一致 |

这些是产品假设，不是已经证明的用户结论。

## 目标用户假设

首轮聚焦愿意了解近期状态、偶尔尝试记录，但因为输入负担高或记录后没有反馈而难以持续的成年人。

不预设他们：

- 一定喜欢 RPG。
- 一定存在时间焦虑。
- 一定愿意分享生活。
- 一定信任 AI。
- 一定需要人生规划。

研究以行为和任务为准，详见[目标用户与待验证任务](../research/user-personas.md)。

## 内容价值闭环

```text
人生进度带来好奇
  → 第一次轻量记录
  → 立即收到具体回应
  → 愿意再记录一次
  → 累积记录形成可核对的个人线索
  → 用户选择想继续观察的问题
```

如果当日回应不能促成第二次记录，不应继续堆叠成就、社交或深度 AI。

## 价值层级

### 第一层：理解

- 一眼看懂产品，不学习复杂 RPG 规则。
- 人生进度明确标为估算，并可随时关闭。
- 清楚数据保存位置，以及 AI 是否会收到内容。

### 第二层：记录

- 60 秒内完成。
- 忘记记录没有惩罚。
- 离线可用，同一天编辑不产生重复记录。
- 数据可以导出、重新导入和清空。

### 第三层：当日回应

- 保存后立即获得一条有依据的本地回应。
- 优先回应明确事件，不把全部字段机械复述一遍。
- 信息足够时最多提出一个开放问题。
- 用户可以标记有帮助、没帮助或事实不准确。
- 可选 AI 只改善语言表达，不创造新事实。

### 第四层：累积发现

- 达到样本门槛后才描述趋势或关联。
- 显示样本数、日期范围、缺失和原始依据。
- 由用户选择想继续观察的问题。
- 这一层属于 v0.2 以后实验，不是 v0.1 已上线能力。

## v0.1 功能矩阵

| 功能 | 用户价值 | 优先级 |
|------|----------|--------|
| 可关闭的估算进度 | 第一眼辨识度和时间掌控感 | P0 |
| 每日记录 | 低成本保存当天状态 | P0 |
| 本地当日回应 | 保存后立即获得具体反馈 | P0 |
| 回应质量反馈 | 纠正内容并验证是否有用 | P0 |
| 可选 AI 文字整理 | 在事实不变的前提下改善表达 | P0 实验 |
| 历史和导入导出 | 用户拥有并控制自己的数据 | P0 |
| 中性记录统计 | 提升回看感，不制造惩罚 | P1 |
| 周期证据、主动探索、月度叙事 | 形成长期内容价值 | 后续验证 |
| 排行榜、诊断和重大决策 AI | 高风险且偏离核心 | 不做 |

## 伦理与体验原则

- **自主而非依赖**：帮助用户观察自己，不让用户服从 AI。
- **成长而非寿命竞赛**：不公开比较年龄、寿命进度或人生阶段。
- **事实而非伪科学**：没有足够样本时明确说不知道。
- **回应而非评判**：描述记录中的事实，不定义用户人格。
- **鼓励而不操控**：无断签惩罚、羞辱、紧迫提醒或付费恢复。
- **隐私默认**：本地优先；AI 和行为分析分别取得明示同意。
- **内容可追溯**：长期结论必须能回到支撑记录。
- **包容人生路径**：不把年龄等同于人生任务。

## 成功定义

v0.1 成功不是停留时长或 AI 使用次数，而是：

- 用户能低负担地留下真实记录。
- 用户能指出当日回应依据了什么。
- 回应被认为具体、有帮助且不过界。
- 用户愿意在 7 日内再记录一次。
- 拒绝 AI 的用户仍然获得完整核心价值。
- 产品没有显著增加焦虑、比较或隐私压力。

长期成功还需要证明：

- 累积内容比普通统计图更容易理解。
- 用户可以查看证据、纠正结论并自主选择观察方向。
- 产品价值随个人记录积累而增加，而不是靠提醒和奖励维持。
