# 人生进度条 RPG — 用户体验设计文档 v2.0 (UX Spec)

> 版本：v2.0 | 日期：2026-07-23
> 核心原则：**爽感让你打开，价值让你留下**

> 内容行为以[内容价值与个人分析策略](./content-strategy.md)为准。实施说明：本文包含完整产品愿景。v0.1 只实施 [MVP 规划](../planning/milestone-v1.md) 中的 P0/P1；成就、XP、分享、报告和深度 AI 均为后续方向。统一口径：出生年份、心情 1～5、能量 0～10、进度可关闭、AI 可选。

---

## 一、产品定位

### 1.1 一句话定义
用人生进度建立辨识度，用轻量记录降低门槛，用有依据的回应和累积发现创造长期价值。

### 1.2 核心体验锚点

| 锚点 | 目标 | 设计要点 |
|------|------|----------|
| 第一感 | "哇，好酷！" | 3秒内产生震撼 |
| 第一用 | "我也要！" | 30秒内完成首次记录 |
| 第一天 | “这次记录有回应” | 首次看到具体当日回应 |
| 第一周 | “这些记录开始有线索” | 达到门槛后查看周期证据 |
| 第一月 | “这些片段值得保存” | 用户主动生成月度回顾 |

### 1.3 爽感 vs 价值 双轮驱动

| 维度 | 爽感层 | 价值层 |
|------|--------|--------|
| 驱动力 | 多巴胺、好奇心、社交炫耀 | 自我成长、掌控感、人生意义 |
| 表现形式 | 克制的进度反馈和轻动效 | 当日回应、周期证据、主动观察问题 |
| 用户感受 | "好酷！" "我要发朋友圈" | "原来我是这样的" "我知道该怎么做了" |
| 留存作用 | 短期吸引 | 长期信任和回看价值 |
| 时机 | 打开 App 前 3 秒 | 使用 App 后 30 秒+ |

### 2.3 组件状态定义

#### 按钮状态

| 状态 | 视觉表现 | 交互行为 |
|------|----------|----------|
| Default | 主题色背景，白色文字 | 可点击 |
| Hover | 亮度 +10%，轻微放大 | 鼠标悬停 |
| Active | 缩小 95%，暗 5% | 按下反馈 |
| Loading | 旋转图标，文字变为"存档中..." | 禁用点击 |
| Disabled | 灰色背景，禁止光标 | 完全禁用 |
| Success | 绿色背景，✓ 图标 | 反馈后消失 |

#### 卡片状态

| 状态 | 视觉表现 | 交互行为 |
|------|----------|----------|
| Default | 白色背景，轻阴影 | 可点击 |
| Hover | 上浮 2px，阴影加深 | 暗示可交互 |
| Active | 下沉 1px | 按下反馈 |
| Selected | 主题色边框 | 表示选中 |
| Disabled | 50% 透明度 | 不可交互 |
| Loading | 骨架屏动画 | 内容加载中 |
| Error | 红色边框，错误图标 | 显示错误状态 |

#### 输入框状态

| 状态 | 视觉表现 | 交互行为 |
|------|----------|----------|
| Default | 灰色边框 | 等待输入 |
| Focus | 主题色边框，发光效果 | 光标闪烁 |
| Filled | 主题色边框 | 有内容 |
| Error | 红色边框，错误提示 | 显示错误 |
| Disabled | 灰色背景 | 不可编辑 |

## 四、AI 价值层级

| 层级 | 能力 | 示例 | 价值感 |
|------|------|------|--------|
| L1 记录总结 | 总结当日记录 | "今天能量值 7，心情不错" | ⭐⭐ |
| L2 趋势分析 | 对比本周 vs 上周 | "本周心情比上周提升 0.5" | ⭐⭐⭐ |
| L3 关联探索（后续） | 显示样本量，明确相关不等于因果 | "12 条记录中，运动标签日的平均心情较高" | 待验证 |
| 预测、诊断、人生规划 | 证据与安全风险过高 | 不提供 | 禁止 |

## 三、价值体系设计

### 3.1 价值输出矩阵

| 时间维度 | 价值内容 | 输出形式 |
|---------|---------|---------|
| 每日 | 有依据的当日回应 | 记录保存后显示，本地默认、AI 可选 |
| 每周 | 周期证据：覆盖情况 + 达到门槛的关联线索 + 观察问题 | 用户主动查看 |
| 每月 | 可编辑的月度片段：事实 + 变化 + 不确定性 | 用户主动生成 |
| 每季 | 季报：季度主题总结 + 人生阶段评估 | 章节切换时推送 |
| 每年 | 年度史诗：全年关键词 + 人生转折点 + 新年建议 | 年底自动生成 |

## 四、用户成长路径

```
Day 1：好奇打开 → 看到进度条 → "哇，好酷"（爽感）
   ↓
Day 2-7：每天记录 → 看到 AI 反馈 → "原来我是这样的"（价值初现）
   ↓
Day 8-30：记录达到门槛 → 主动查看周期证据 → 选择一个观察问题（价值确认）
   ↓
Day 31-90：累积真实片段 → 主动生成月度回顾 → 用户确认或修正自己的解释（价值深化）
   ↓
Day 91+：自主回顾 → 用户能结合记录形成自己的判断（价值稳定）
```

## 五、爽感与价值的平衡设计

### 5.1 交互节奏设计

```
用户操作 → 爽感即时反馈（0-1秒）→ 价值延迟呈现（1-3秒）
   ↓
例：点击"存档今日"
   ↓
0ms：按钮按下动画（爽感）
500ms：✅弹出 + 震动 + +1经验值（爽感）
1000ms：🔥连续天数更新（爽感）
1500ms：AI 洞察气泡滑入（价值）
2000ms：数据图表展开（价值）
```

### 5.2 文案双轨设计

| 场景 | 爽感文案 | 价值文案 |
|------|---------|---------|
| 记录完成 | "+1 经验值！存档成功！" | "你的数据又丰富了一点，AI 将给出更精准的建议" |
| 徽章解锁 | "🏆 解锁新成就！超越 73% 用户！" | "连续记录 7 天，你的平均心情提升了 0.8 分" |
| 章节切换 | "🎉 进入新章节！等级提升！" | "基于你的数据，新章节建议重点关注..." |
| 每日打开 | "今天是你在地球的第 X 天！" | "本周目标完成度 67%，再坚持 2 天解锁..." |

## 六、错误处理与边界状态

### 5.1 网络异常处理

| 场景 | 用户感知 | 视觉表现 | 后续行为 |
|------|----------|----------|----------|
| 网络断开 | 无感 | 静默重试 3 次 | 后台自动保存本地 |
| 重试失败 | 轻提示 | Toast: "网络不稳定，已本地保存" | 显示重试按钮 |
| 网络恢复 | 无感 | 自动同步 | 成功后无提示 |

### 5.2 空状态设计

| 页面 | 空状态文案 | 视觉表现 | CTA 按钮 |
|------|------------|----------|----------|
| 首页-无记录 | "今天值得被记录，开始你的冒险吧" | 渐变背景 + 问号图标 | "记录今天" |
| 成就页 | "成就殿堂空空如也，快去解锁第一个吧" | 锁图标 + 问号 | "去探索" |
| AI 对话 | "人生向导正在等你，说点什么吧" | 对话气泡 | "开始对话" |
| 数据页 | "记录 7 天后解锁数据分析" | 进度条 + 锁 | "去记录" |

### 5.3 加载状态设计

| 场景 | 加载表现 | 时长 | 预期 |
|------|----------|------|------|
| 首页加载 | 骨架屏 | < 1s | 即时 |
| 记录提交 | 按钮 Loading | 1-3s | 感知进度 |
| AI 生成 | 打字效果 | 2-5s | 真实生成感 |

## 九、竞品差异化（爽感 + 价值）

| 维度 | 现有工具 | 人生进度条 RPG |
|------|---------|-----------|
| 情感基调 | 焦虑（死亡倒计时） | **爽感 + 成长（RPG 升级）** |

| 数据价值 | 无 | **有证据的个人回顾 + 用户主动探索** |
| 个性化 | 无 | **章节系统 + 个人数据驱动的定制化** |
| 长期价值 | 无 | **人生档案 + 年度史诗 + 成长轨迹** |
