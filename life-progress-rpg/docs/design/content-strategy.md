# 内容价值与个人分析策略

> 状态：产品内容真源  
> 适用范围：每日复盘、趋势、周期回顾、AI 输出和相关产品文案  
> 版本边界：v0.1 只实现每日回应；周期发现和主动探索按路线图验证后开放

## 1. 产品内容命题

产品不是“带进度条的日记”，也不是替用户解释人生的 AI 导师。

核心承诺是：

> **让一次轻量记录立即得到具体回应，并让累积记录逐渐形成可核对、可理解、由用户掌控的个人线索。**

各能力承担不同职责：

| 能力 | 职责 | 不能承担的职责 |
|------|------|----------------|
| 估算人生进度 | 制造第一眼辨识度和时间感知 | 不能成为焦虑倒计时或留存惩罚 |
| 低负担记录 | 降低输入成本，积累真实个人素材 | 不能只是收集数据 |
| 当日回应 | 让第一次记录就有结果 | 不能伪装成长期洞察 |
| 周期发现 | 帮助用户回看变化和可能的关联 | 不能把相关说成因果 |
| 主动探索 | 帮用户验证自己关心的问题 | 不能替用户做决定 |
| 美观呈现 | 提升理解、信任和分享意愿 | 不能用视觉包装空洞结论 |
| 本地优先与导出 | 建立长期信任和数据掌控感 | 不能只作为营销口号 |

## 2. 用户价值闭环

```text
人生进度带来好奇
  → 60 秒完成一次真实记录
  → 马上收到具体、克制的回应
  → 累积足够记录后发现可核对的个人模式
  → 用户选择下一步想观察的问题
  → 新记录变得更有目的
```

如果产品只完成“好奇 → 记录”，用户容易体验一次后离开。如果反馈只是随机模板或泛化鼓励，记录越多也不会增加价值。

## 3. 内容价值阶梯

### L0：首次理解

**触发**：首次打开。  
**内容**：估算进度说明、产品价值和隐私边界。  
**成功信号**：用户理解“估算、可关闭、本地优先”，并愿意开始第一次记录。

L0 是获客钩子，不是长期卖点。

### L1：当日回应

**触发**：每次记录保存后。  
**版本**：v0.1。  
**内容结构**：

1. 一条基于本次输入的事实回应。
2. 信息足够时，最多一个温和、开放的问题。
3. 明确标记来源为“本地回应”或“AI 整理”。

示例：

```text
今天完成了项目评审，心情较轻松。看起来这件事对你很重要。
```

```text
今天能量只有 3/10，你仍然记录了下来。此刻更需要休息，还是减少一件事？
```

当用户只选择了数值、没有备注或标签时：

```text
今天是平静的一天。记录还很少，暂时不判断趋势。
```

L1 不能声称发现模式，不能把字段机械复述一遍，也不能强行提出问题。

### L2：周期发现

**触发**：达到最低数据门槛且用户主动查看。  
**版本**：v0.2 候选实验。  
**内容**：

- 本周发生了什么
- 哪些变化有记录支持
- 哪些只是值得继续观察的线索
- 一个由用户选择的下周观察问题

示例：

```text
过去 7 天记录了 5 天。有“运动”标签的 3 天，平均能量为 7.0；
其他 2 天为 5.5。样本还少，这只是当前记录中的关联。
```

必须同时显示样本数、日期范围、缺失情况和支撑记录入口。

### L3：月度叙事

**触发**：周期内数据达到门槛且用户主动生成。  
**版本**：后续研究。  
**内容**：

1. 本月值得记住的事实。
2. 状态变化和重复出现的主题。
3. 用户曾选择观察的问题及结果。
4. 尚不能判断的部分。
5. 可由用户编辑、保存或删除的个人总结。

月度叙事不是自动生成的人格评价，也不使用“你的本质是”“你总是”等固定标签。

### L4：主动探索

**触发**：用户主动选择一个问题。  
**版本**：后续研究。  
**示例问题**：

- 有运动的日子，我的能量是否通常更高？
- 晚睡之后，第二天的心情有什么变化？
- 哪类活动之后，我更愿意记录积极事件？

产品只帮助设计记录范围、汇总结果和展示不确定性。不得把观察性记录包装成医学实验或因果证明。

## 4. 内容单元规范

周期分析内容至少包含：

| 字段 | 作用 |
|------|------|
| `title` | 用事实主题命名，不使用夸张结论 |
| `observation` | 描述记录中真实出现的变化或关联 |
| `evidence` | 样本数、日期范围、分组值和原始记录入口 |
| `uncertainty` | 缺失、样本不足和“相关不代表因果” |
| `nextQuestion` | 可选的观察问题，不是命令 |
| `controls` | 有帮助/没帮助、纠正、删除和关闭入口 |

每日回应只要求 `observation`，可选 `nextQuestion`；不得为了填满卡片生成没有价值的字段。

## 5. 分析门槛

以下是产品展示门槛，不代表统计学证明：

| 输出 | 最低门槛 | 不足时的行为 |
|------|----------|--------------|
| 当日回应 | 1 条有效记录 | 只回应当前事实 |
| 7 日回顾 | 过去 7 天至少记录 5 天 | 展示覆盖情况，不输出模式 |
| 两组比较 | 每组至少 3 条，总计至少 8 条 | 提示继续记录 |
| 方向性趋势 | 至少 14 条，跨度至少 21 天 | 不显示趋势结论 |
| 月度叙事 | 30 天内至少记录 18 天 | 生成“本月片段”，不称完整月报 |

所有分析都必须：

- 使用用户本地日期。
- 显示实际样本数和日期范围。
- 允许查看支撑结论的原始记录。
- 区分“事实”“关联线索”“用户自己的解释”。
- 不用插值伪造缺失日期。
- 不跨用户比较，不显示“超过 X% 用户”。
- 不根据出生年份推断人生阶段、能力或目标。

## 6. 当日回应质量

### 6.1 优先级

按以下顺序选择最值得回应的内容：

1. 用户备注中明确发生的事件。
2. 用户主动选择的标签。
3. 心情和能量之间明显但仅限当日的组合。
4. 对“完成记录”本身的中性确认。

不要每次同时复述心情、能量、标签和备注。

### 6.2 变化与防重复

- 连续 3 次回应不得使用相同开头。
- “今天”“看起来”“也许”不能成为固定模板前缀。
- 同一条记录重新生成时应保留事实一致性，但允许表达变化。
- 没有新信息时宁可简短，也不填充鸡汤。
- 本地规则库按事件、精力、休息、关系、成长和一般记录分类，但不做人格分类。

### 6.3 禁止内容

- “太棒了，坚持就是胜利”
- “你正在超越大多数人”
- “AI 发现你是一个……的人”
- “这证明运动让你更快乐”
- “明天必须继续打卡”
- “你的人生已经过去了 X%，不要浪费时间”

## 7. 个性化与用户控制

个性化来自用户明确提供和允许使用的数据，而不是推测敏感属性。

用户必须能够：

- 关闭人生进度但保留记录功能。
- 分别关闭 AI、行为分析和周期回顾。
- 查看一条结论使用了哪些记录。
- 将回应标记为有帮助、没帮助或事实不准确。
- 删除单条回应，不删除原始记录。
- 导出原始记录及生成内容。

“没帮助”和“事实不准确”不能只做埋点，必须进入后续质量评估；未经同意不得把正文发送到分析服务。

## 8. AI 与本地规则的分工

本地代码负责：

- 样本门槛
- 日期范围与分组计算
- 缺失数据
- 数值统计
- 可追溯证据
- 确定性失败回退

AI 只负责：

- 把已经计算出的事实组织成自然中文
- 在不增加新事实的前提下压缩和排序
- 生成一个可选的开放式观察问题

禁止让模型自行读取原始记录后计算平均值、相关性或趋势。先由可测试代码得出结构化事实，再让模型表述。

## 9. 内容效果指标

### v0.1

| 指标 | 目的 |
|------|------|
| 首次回应展开率 | 用户是否愿意阅读结果 |
| 回应有用率 | 内容是否具体且有价值 |
| 回应不准确率 | 事实一致性和越权风险 |
| 7 日内第二次记录率 | 当日回应是否带来再次使用 |
| AI 关闭率 | AI 是否造成不信任或打扰 |

### 周期内容实验

| 指标 | 目的 |
|------|------|
| 周期回顾查看率 | 累积内容是否形成期待 |
| 证据展开率 | 用户是否关心依据 |
| 观察问题选择率 | 内容是否促成自主探索 |
| 回顾后记录率 | 回顾是否增加后续记录目的 |
| 焦虑/冒犯反馈率 | 内容是否带来负面影响 |

停留时长、AI 对话轮数和“离不开产品”不是成功指标。

## 10. 验证方法

上线前至少用以下记录集评测：

- 只有数值，没有文字
- 正向事件但低能量
- 负向事件但高能量
- 连续多天内容相似
- 长备注、错别字和口语
- 缺失日期
- 标签与正文表面冲突
- 敏感、危机或求助内容

人工评审逐条判断：

1. 是否完全有依据。
2. 是否说出了用户一眼可见之外的整理价值。
3. 是否承认数据不足。
4. 是否自然、简洁且不重复。
5. 是否避免诊断、因果、预测和操控。
6. 用户能否理解并纠正它。
