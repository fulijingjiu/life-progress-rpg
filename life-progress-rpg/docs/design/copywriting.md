# 文案与内容表达指南

> 状态：v0.1 用户可见文案规范
> 最近校准：2026-07-24
> 内容结构和分析门槛以[内容价值与个人分析策略](./content-strategy.md)为准

## 1. 产品声音

产品像一位尊重边界的记录整理者：

- **温和**，但不自动乐观。
- **具体**，但不擅自解释。
- **诚实**，会说不知道。
- **平等**，不命令、不训诫。
- **克制**，不用感叹号和 Emoji 制造价值。

不要把 AI 写成朋友、导师、心理咨询师或人格化陪伴对象。

## 2. 基本写法

| 场景 | 应该表达 | 避免 |
|------|----------|------|
| 标题 | 用户在哪里 | 营销口号 |
| 按钮 | 将执行什么动作 | “好的”“继续看看” |
| 辅助文字 | 动作结果和数据去向 | 重复标题 |
| 成功 | 已完成的事实 | 夸张庆祝 |
| 错误 | 什么没完成、如何恢复 | 把错误叫“新发现” |
| 回应 | 本次记录里最具体的事实 | 人格评价和鸡汤 |
| 数据不足 | 还缺什么 | 用模糊洞察填满 |
| 隐私 | 发送什么、给谁、为什么 | “请放心”“绝对安全” |

规则：

- 按钮优先使用动宾短语。
- 一句话只承担一个任务。
- Placeholder 只示例输入，不能代替 Label。
- 不回避“删除、错误、失败”等准确词。
- 不用“旅程、冒险、能量”等隐喻覆盖所有界面。
- 正文通常使用句号；非庆祝场景不使用感叹号。

## 3. 术语表

| 统一使用 | 不使用 |
|----------|--------|
| 估算人生进度 | 寿命倒计时、剩余人生 |
| 当日回应 | AI 人生洞察、命运提示 |
| 周期回顾 | 权威周报、人生报告 |
| 观察线索 | 诊断、结论、规律 |
| 记录天数 | 连胜、断签 |
| 删除记录 | 清除失败、重新开始 |
| AI 文字整理 | AI 人生导师、AI 更懂你 |
| 事实不准确 | AI 犯错了（作为按钮标签过长） |

“心情”和“能量”是用户输入字段，不称为心理健康评分。

## 4. 首次引导

### 价值说明

标题：

```text
留下一点今天，以后更容易看见变化
```

正文：

```text
每天用不到一分钟记录心情、精力和发生的事。记录先保存在此设备。
```

主按钮：

```text
开始设置
```

### 出生年份

Label：

```text
出生年份
```

辅助文字：

```text
只用于计算估算进度。它不是寿命预测，你可以随时关闭显示。
```

开关：

```text
显示估算人生进度
```

### AI 同意

标题：

```text
是否启用 AI 文字整理？
```

说明模板：

```text
启用后，本次记录的心情、能量、标签和备注会发送给 {供应商}，
用于生成一条当日回应。记录功能和本地回应不需要启用 AI。
```

按钮：

```text
暂不启用
启用 AI 整理
```

不能使用：

```text
开启更懂你的 AI
解锁完整体验
```

## 5. 首页

### 未记录

标题：

```text
今天还没有记录
```

正文：

```text
选一下此刻的状态，也可以补充今天发生的事。
```

主按钮：

```text
记录今天
```

### 已记录

标题：

```text
今天已记录
```

辅助文字：

```text
最后更新于 {time}
```

按钮：

```text
编辑今天
```

不使用“连续第 X 天，别停下来”。

### 进度

标题：

```text
估算人生进度
```

说明：

```text
根据出生年份和你设置的预期年限粗略计算，不代表寿命预测。
```

关闭后不出现暗示年龄或剩余时间的替代文案。

## 6. 每日记录

字段：

```text
今天的心情
今天的能量
发生了什么（可选）
标签（可选）
```

备注 Placeholder：

```text
例如：完成了项目评审，晚上散了会儿步
```

保存按钮：

```text
保存记录
```

更新按钮：

```text
更新今天
```

未保存离开：

```text
这次修改还没有保存。
```

按钮：

```text
继续编辑
放弃修改
```

## 7. 当日回应

### 结构

1. 优先回应备注中的明确事件。
2. 没有事件时选择一个标签或当日状态。
3. 信息足够时最多提出一个开放问题。
4. 只有当日数据时不使用“趋势、经常、总是、通常”。

合格：

```text
今天完成了项目评审，心情较轻松。看起来这件事对你很重要。
```

```text
今天能量是 3/10，你还是留下了记录。此刻更需要休息，还是少安排一件事？
```

```text
今天的状态已经记下。记录还少，暂时看不出变化。
```

不合格：

```text
太棒了，坚持就是胜利！
```

```text
AI 发现你是一个积极向上的人。
```

```text
运动让你的心情提升了 30%。
```

### 来源标签

```text
本地回应
AI 文字整理
```

不用“AI 深度洞察”。

### 质量反馈

问题：

```text
这条回应有帮助吗？
```

选项：

```text
有帮助
没帮助
事实不准确
```

提交后：

```text
反馈已保存在此设备
```

如果获得行为分析同意并确实上传匿名事件：

```text
已记录你的选择，不包含记录正文
```

## 8. AI 状态与错误

生成中：

```text
正在整理这次记录…
```

超时：

```text
记录已保存在本地。AI 整理暂时没有完成，当前显示本地回应。
```

按钮：

```text
稍后重试
```

未同意：

```text
当前使用本地回应，不会发送记录内容。
```

输出被拦截：

```text
这次没有显示 AI 回应。你的原始记录已安全保存在本地。
```

不要显示供应商原始错误、Prompt 或安全规则细节。

## 9. 历史与数据

空状态：

```text
还没有历史记录
完成第一次记录后，可以在这里回看和编辑。
```

按钮：

```text
记录今天
```

删除确认：

```text
删除 {date} 的记录？
删除后无法恢复。其他日期不会受影响。
```

按钮：

```text
取消
删除记录
```

导出成功：

```text
数据文件已生成
```

导入预览：

```text
文件包含 {records} 条记录和 {settings} 份设置。确认后将写入此设备。
```

导入失败：

```text
无法导入这个文件。现有数据没有改变。
原因：{reason}
```

清空确认：

```text
清空此设备上的全部数据？
这会删除设置、记录和生成的回应，且无法恢复。建议先导出备份。
```

## 10. 周期内容（后续实验）

未达到门槛：

```text
过去 7 天记录了 3 天。再积累一些记录后，才能比较不同日子的状态。
```

达到门槛：

```text
过去 7 天记录了 5 天。有“运动”标签的 3 天，平均能量为 7.0；
其他 2 天为 5.5。样本还少，这只是当前记录中的关联。
```

操作：

```text
查看依据
选择观察问题
这条没帮助
```

禁止：

- 自动使用“你应该”给出下周任务。
- 把缺失日期当作低心情或未完成。
- 用“解锁第一份周报”制造损失压力。
- 使用“超越 X% 用户”。

## 11. 敏感内容

用户表达明显危险或求助时：

- 先承认内容值得认真对待。
- 鼓励联系现实中的可信任的人、当地紧急服务或合格专业人员。
- 不诊断，不用游戏奖励，不转移到连续记录。
- 热线信息必须通过可靠地区来源查询，不猜测号码。

通用开头：

```text
你写下的情况听起来需要现实中的支持。请优先联系身边可信任的人，
或当地紧急服务和合格专业人员。
```

实际发布前必须完成地区化危机文案评审。

## 12. 发布检查

- 同一概念是否全产品使用同一名称？
- 按钮是否说明真实动作？
- 成功文案是否说明保存位置？
- 错误文案是否给出恢复路径？
- 回应是否能在输入中找到依据？
- 数据不足时是否明确说不知道？
- 是否存在鸡汤、羞辱、连续压力、跨用户比较或人格判断？
- AI、分析和隐私说明是否与真实实现一致？
- 长文案、字体放大和窄屏下是否仍可理解？
