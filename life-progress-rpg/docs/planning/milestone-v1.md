# MVP 规划 v0.1

> 状态：当前唯一实施基线
> 最近校准：2026-07-24

## 版本目标

验证“估算进度带来第一次注意 + 低负担记录 + 具体当日回应”能否促成第二次记录，而不是验证完整 RPG、社交、周期报告或 AI 人生规划。

### 核心假设

1. 用户能在 60 秒内完成首次记录。
2. 可关闭、明确标注为估算的人生进度带来掌控感，而不是明显焦虑。
3. 用户能指出当日回应的事实依据，并认为它比机械复述或泛化鼓励更有价值。
4. 一次具体回应能提高用户在 7 日内第二次记录的意愿。

### 非目标

v0.1 不实现账号、云同步、多设备同步、公开分享、排行榜、正式成就/XP、推送、周报/月报、深度 AI 对话、人生规划和重大决策建议。规划中的能力不得在 v0.1 用户指南中表述为已上线。

## 功能范围

### P0：必须完成

| 功能 | 要求 | 估算 |
|------|------|------|
| 首次引导 | 昵称可选；收集出生年份；解释估算口径；可关闭进度展示 | 4h |
| 估算进度 | 出生年份 + 可调预期年限；始终标注“估算” | 4h |
| 每日记录 | 心情 1～5、能量 0～10、标签、可选备注 | 6h |
| 当日回应 | 本地规则立即生成；可选 AI 只整理语言；支持有帮助/没帮助/事实不准确反馈 | 7h |
| 历史记录 | 查看、编辑、删除；每个本地日期最多一条 | 4h |
| 数据管理 | JSON 导出、导入、清空和二次确认 | 4h |
| 设置 | 昵称、出生年份、预期年限、进度显示、主题、AI 同意 | 3h |
| 异常与隐私 | 离线可保存；AI 失败不影响记录；明确隐私说明 | 4h |

### P1：有余量再做

| 功能 | 要求 |
|------|------|
| 连续记录 | 只做中性统计，不设置断签惩罚 |
| 进度动画 | 支持 `prefers-reduced-motion` |

## 统一数据模型

```typescript
interface UserSettings {
  id: string;
  nickname?: string;
  birthdayYear: number;
  lifeExpectancy: number;
  showLifeProgress: boolean;
  aiConsent: boolean;
  analyticsConsent: boolean;
  theme: 'default' | 'spring' | 'study' | 'explore' | 'root';
  createdAt: string;
  updatedAt: string;
}

interface LifeRecord {
  id: string;
  userId: string;
  localDate: string; // 用户本地时区 YYYY-MM-DD
  mood: 1 | 2 | 3 | 4 | 5;
  energy: number; // 整数 0～10
  content?: string;
  tags: string[];
  reflection?: string;
  reflectionSource: 'rules' | 'ai' | 'none';
  reflectionStatus: 'not_requested' | 'pending' | 'completed' | 'failed';
  reflectionFeedback?: 'helpful' | 'not_helpful' | 'inaccurate';
  createdAt: string;
  updatedAt: string;
}
```

约束：

- 只收集出生年份，因此不能宣称精确已活天数。
- 真实 AI 请求必须经过服务端代理，浏览器不得保存供应商 API Key。
- 未获得 AI 明示同意时，不向第三方发送记录内容。
- 日志和埋点不得包含昵称、日记正文或 AI 对话原文。

## 页面与关键流程

```text
首次引导 → 首页 → 记录抽屉 → 本地保存成功
                                  ├─ 本地当日回应（默认）
                                  ├─ AI 文字整理（已同意且代理可用）
                                  └─ 回应质量反馈（仅本地保存）

首页 → 历史 → 查看/编辑/删除
首页 → 设置 → 隐私/显示偏好/数据导入导出
```

## 交付节奏

| 阶段 | 内容 |
|------|------|
| Day 1～2 | 工程、数据模型、本地日期、校验与迁移 |
| Day 3 | 引导、估算说明、显示开关 |
| Day 4～5 | 首页、记录、历史 |
| Day 6 | 当日回应、质量反馈、可选 AI 代理与失败回退 |
| Day 7～8 | 设置、编辑删除、导入导出 |
| Day 9～10 | 无障碍、异常、数据迁移、端到端测试 |

## 验收标准

### 功能

| 项目 | 验收条件 |
|------|----------|
| 引导 | 60 秒内完成；昵称可跳过；进度可关闭 |
| 估算进度 | 明确显示“估算”；设置变化后正确重算 |
| 记录 | 3 步内完成；离线可保存；同日编辑不产生重复记录 |
| 当日回应 | 立即显示且能指出输入依据；不声称趋势；AI 失败不阻塞保存；可提交三类质量反馈 |
| 数据控制 | 数据能导出、重新导入和清空；清空前二次确认 |
| 隐私 | 未同意时不调用 AI；前端构建产物中不存在 API Key |

### 质量

| 指标 | 标准 |
|------|------|
| LCP | 常规 4G、生产构建下 < 2.5s |
| INP | < 200ms |
| 界面与内容 | 所有用户可见页面满足[设计质量基线](../design/quality-bar.md)；关键视口、核心状态、交互反馈和中文文案已复核 |
| 可访问性 | 键盘可操作；正文对比度 ≥ 4.5:1；支持 Reduce Motion |
| 稳定性 | 核心流程无未处理异常；损坏导入数据有明确错误 |

### 产品验证

埋点必须取得明示同意。以下是初始判断线，不是已验证结论：

| 指标 | 定义 | 判断线 |
|------|------|--------|
| 引导完成率 | 开始引导后完成设置的独立用户比例 | ≥ 60% |
| 首次记录率 | 完成引导后 24 小时内创建记录的比例 | ≥ 40% |
| 7 日内第二次记录率 | 首次记录后 7 天内再次保存记录的用户比例 | ≥ 30% |
| 回应有用率 | 对已评价回应选择“有帮助”的比例 | ≥ 50% |
| 回应不准确率 | 对已评价回应选择“事实不准确”的比例 | ≥ 10% 时暂停 AI 放量并复查 |
| 焦虑停止线 | 表示进度造成明显焦虑的测试参与者比例 | ≥ 20% 时暂停放量并调整 |

小样本只报告原始人数和访谈结论，不展示看似精确的百分比。

## 风险与对策

| 风险 | 对策 |
|------|------|
| API Key 泄露 | 真实 AI 只走服务端代理；密钥只存在服务端 |
| AI 延迟/失败 | 默认规则模板；异步生成；失败不影响保存 |
| 进度引发焦虑 | 首次说明、显示开关、负面反馈入口、研究停止线 |
| 数据丢失 | JSON 导入导出、迁移测试、清空二次确认 |
| 隐私泄露 | 数据最小化、明示同意、日志脱敏、不记录正文 |
| 范围失控 | P0 未验收前不启动账号、社交、成就和深度 AI |
