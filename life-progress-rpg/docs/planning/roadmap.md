# 产品路线图

> 路线图以内容价值和验证门槛推进，不以固定日期或功能数量推进

## 原则

1. 人生进度负责首次辨识，当日回应负责价值确认，累积分析负责长期价值。
2. 先证明上一层内容有用，再开放下一层；不靠更多功能掩盖内容质量不足。
3. 本地代码负责计算和证据，AI 只整理语言。
4. 不以依赖、停留时长、AI 对话轮数或连续签到压力为目标。
5. 账号、同步、社交和商业化不能先于核心内容闭环。

详细内容模型见[内容价值与个人分析策略](../design/content-strategy.md)。

## v0.1：当日价值验证

**目标**

验证“估算进度带来好奇、60 秒记录降低门槛、具体当日回应促成第二次记录”。

**当前决策状态：**未发布。实现功能闭环已较完整，但仍待阶段 H 人工验收与用户验证指标确认。

**范围**

- 可关闭并明确标注为估算的人生进度
- 心情 1～5、能量 0～10、标签和可选备注
- 本地历史、编辑删除和 JSON 导入导出
- 默认本地回应、经同意的可选 AI 整理
- 回应有帮助/没帮助/不准确反馈
- 隐私同意、离线、失败回退和基础无障碍

**内容门槛**

- 只回应当前记录，不声称发现趋势。
- 优先回应明确事件，不机械复述全部字段。
- 信息足够时最多提出一个开放问题。
- AI 失败不影响保存，并回退到本地回应。

**进入下一阶段前**

- [ ] P0 功能、质量和隐私验收全部通过
- [ ] 完成至少 12 位目标用户的首次记录和内容盲测
- [ ] 多数测试者能指出回应所依据的输入
- [ ] 回应有用率达到 MVP 判断线
- [ ] 7 日内第二次记录率达到判断线
- [ ] 不准确、越权和焦虑停止线未触发，或已调整并复测

## v0.2：周期证据实验

**目标**

验证累积记录能否产生普通情绪打卡没有的回看价值。

**候选能力**

- 7 日覆盖情况与记录摘要
- 达到门槛后展示带样本数的个人关联线索
- 每条结论展开查看支撑记录
- 用户反馈、有帮助、没帮助、事实不准确和删除
- 用户选择一个下周观察问题
- 中性记录统计，可隐藏且无断签惩罚

**明确排除**

- 自动推送和强制周报
- 相关性冒充因果
- “超过 X% 用户”和跨用户比较
- 人格标签、心理诊断和行动命令
- 正式 XP、付费恢复连续天数

**进入下一阶段前**

- [ ] 周期内容满足样本、日期、缺失和证据门槛
- [ ] 用户能正确理解“事实、线索和不知道”的区别
- [ ] 证据展开提高信任而没有造成明显理解负担
- [ ] 回顾后记录率优于只展示图表的对照方案

## v0.3：主动探索实验

**目标**

验证用户是否愿意围绕自己选择的问题进行一段时间的记录和回顾。

**候选能力**

- 从模板选择或自定义一个观察问题
- 说明需要记录的字段、时长和最低样本
- 结构化计算两组差异或方向性变化
- 显示不能判断的原因
- 用户为结果添加自己的解释

**约束**

- 不称为医学实验或因果证明。
- 不要求用户填写健康诊断等敏感字段。
- 不替用户做重大决定。
- 用户随时暂停，已有记录仍可导出。

**进入下一阶段前**

- [ ] 观察问题选择率和完成率达到预设门槛
- [ ] 用户认为结果比凭印象回忆更清楚
- [ ] 没有把弱关联当成确定结论的系统性误解

## v0.4：月度叙事与长期档案

**目标**

验证事实、趋势和用户解释能否形成值得保存的个人档案。

**候选能力**

- 数据充分时生成月度回顾
- 数据不足时生成“本月片段”
- 用户编辑、确认、删除和导出
- 用户自定义人生章节
- 主动生成且默认隐藏敏感字段的分享卡片

月度叙事不自动评价人格、人生阶段或成败。

## 可选账号与同步

只有当研究证明“无法跨设备”是主要流失原因时单独立项。

前置条件：

- 完成隐私、安全和威胁建模评审
- 明确处理目的、供应商、保留期、导出和删除 SLA
- 本地核心功能不因拒绝账号而降级
- 同步冲突和恢复方案经过测试

账号和同步不绑定某个内容版本，也不默认早于内容价值验证。

## 稳定发布

稳定版至少要求：

- 当日、周期和主动探索中实际开放的层级均有质量证据
- 核心流程性能、无障碍和错误率达标
- 数据迁移与恢复经过演练
- AI 安全评测、供应商切换和本地降级可用
- 用户指南只描述实际发布能力
- 隐私说明和产品行为一致

## 永久高风险边界

以下能力不因版本推进自动获得批准：

- 医疗或心理诊断
- 寿命、人生状态或行为结果预测
- 重大决策代理
- 公开人生进度排行
- 未经请求的情感依赖设计
- 利用羞辱、断签损失或焦虑推动付费
