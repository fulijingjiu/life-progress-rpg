## 统一口径

- v0.1 是本地优先的 Web/PWA 验证版，不包含账号、云同步、排行榜、公开社交、正式成就系统和深度 AI 人生规划。
- 出生信息默认只收集年份；因此只展示“估算年龄/进度”，不宣称精确已活天数。用户可关闭人生进度展示。
- 心情统一为 1～5，能量统一为 0～10。
- AI 是可选的记录整理与反思工具，不替代医疗、心理、法律、财务建议，也不替用户做重大决定。
- 浏览器端不得保存任何 AI 服务密钥；所有真实 AI 请求必须通过服务端代理。
- 未上线或未经验证的能力必须标注“规划中”，不得在用户指南中写成已经提供。

## 文档维护规则

- 每个版本只保留一份实施基线；历史文档必须明确标记“归档”。
- 功能、数据字段或版本范围变化时，同时更新 MVP、UX、数据设计和用户指南。
- 技术栈、目录、状态所有权或依赖方向变化时，同步更新代码结构规范。
- 示例代码必须满足安全基线，不使用真实密钥、弱口令或未经认证的用户标识。
- 市场规模、竞品评论和用户结论必须附来源、日期与样本；无证据内容统一称为“待验证假设”。
- 合并前运行 Markdown 链接检查，禁止新增失效的本地链接。
