# 人生进度条 RPG — 技术方案文档

> 版本：v1.0 | 日期：2026-07-23
> 定位：将人生数据游戏化，每一点记录可接入 AI 分析对话

> ⚠️ 归档说明：这是早期方案，不作为实施依据。当前架构、数据和 AI 约束分别见 [architecture.md](./architecture.md)、[database.md](./database.md) 和 [ai-design.md](./ai-design.md)。

---

## 一、技术选型

### 1.1 前端
- **框架**：React 18 + TypeScript
- **样式**：Tailwind CSS + Framer Motion（动画）
- **状态管理**：Zustand（轻量，适合小游戏状态）
- **截图分享**：html2canvas
- **图表**：Recharts（数据可视化）
- **PWA**：vite-plugin-pwa

### 1.2 后端
- **框架**：Node.js + Express / 或 Next.js API Routes
- **AI 接入**：OpenAI API / Claude API / 国产大模型
- **实时通信**：Socket.io（AI 对话流式输出）

### 1.3 数据库
- **主库**：PostgreSQL（结构化数据：用户、记录、成就）
- **缓存**：Redis（会话、排行榜、热点数据）
- **文件存储**：阿里云 OSS / AWS S3（头像、分享卡片）

### 1.4 部署
- **前端**：Vercel
- **后端**：Railway / Render / 阿里云 ECS
- **数据库**：Supabase PostgreSQL / 自建

### 2.2 核心模块

| 模块 | 职责 |
|------|------|
| 用户系统 | 注册/登录/档案管理 |
| 人生引擎 | 计算进度、章节、等级 |
| 记录系统 | 每日记录、事件记录 |
| 成就系统 | 徽章、里程碑、排行榜 |
| AI 对话 | 分析记录、生成建议、情感对话 |
| 分享系统 | 生成卡片、社交传播 |

## 三、数据库设计

### 3.1 用户表 (users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| nickname | VARCHAR(50) | 昵称 |
| birthday_year | SMALLINT | 出生年份（仅用于估算） |
| life_expectancy | INT | 预期寿命（默认80） |
| theme | VARCHAR(20) | 主题色 |
| avatar_url | TEXT | 头像地址 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 3.2 人生记录表 (life_records)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 外键 → users |
| record_date | DATE | 记录日期 |
| content | TEXT | 记录内容 |
| mood | INT | 心情评分 1-5 |
| energy | INT | 能量值 0-10 |
| category | VARCHAR(30) | 分类：工作/学习/健康/关系/娱乐 |
| tags | JSONB | 标签数组 |
| ai_summary | TEXT | AI 生成的当日总结 |
| created_at | TIMESTAMP | 创建时间 |

### 3.3 成就表 (achievements)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 外键 |
| achievement_type | VARCHAR(50) | 成就类型 |
| title | VARCHAR(100) | 成就名称 |
| description | TEXT | 成就描述 |
| unlocked_at | TIMESTAMP | 解锁时间 |
| is_new | BOOLEAN | 是否新获得 |

## 四、游戏化机制设计

### 4.1 核心数值

| 数值 | 来源 | 用途 |
|------|------|------|
| 人生进度 | 生日计算 | 主进度条 |
| 章节进度 | 当前年龄 / 章节区间 | 章节子进度 |
| 能量值 | 每日记录自评 | 影响 AI 对话风格 |
| 连续记录天数 | 行为数据 | 解锁"坚持者"系列徽章 |
| 影响力 | 分享次数 + 互动 | 排行榜排序 |

### 4.2 徽章系统（示例）

| 徽章 | 解锁条件 |
|------|---------|
| 🌱 萌芽者 | 完成首次记录 |
| 🔥 连续7天 | 连续记录7天 |
| 💎 洞察者 | 首次使用 AI 分析 |
| 🚀 分享达人 | 分享卡片被点击10次 |
| 🏆 章节通关 | 进入下一人生章节 |

## 六、风险与对策

| 风险 | 对策 |
|------|------|
| AI 调用成本高 | 限制免费用户次数，付费解锁无限对话 |
| 用户留存低 | 推送每日记录提醒 + 连续记录奖励 |
| 数据隐私敏感 | 生日仅存年份推算，不存精确日期；本地优先存储 |
| 分享传播弱 | 卡片设计必须精美，文案必须扎心 |
